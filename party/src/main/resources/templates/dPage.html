Dynamic page

To illustrate this, we add a variable in the java class HomeController and then use that variable in the home.html page (the view).
To pass data between the Controller and the View we use a Model.

To pass data between the Controller and the View we use a Model.
  • in the HomeController class add a field “applicationName” of type String and initialize it with value "Party!"
  • Modify the Controller method so that the View can use our new field (via the Model):
    • Add a parameter to the home() method with name model and type Model
    • import the suggested packages
    • Spring will see that your method needs the model and will pass it when it calls the method
  • Now add the field applicationName to the model:
    • Use the method addAttribute of Model.
    • The attribute is stored as a key-value-pair. The 2 parameters of addAttribute are the key and the value.
  • Result: 
    •                     
@GetMapping("/")
public String home(Model model) {
    model.addAttribute("appName", applicationName);
    return "home";
}
Now you can activate Thymeleaf for that html page:
  • We use Thymeleaf as a template engine. Thymeleaf makes it possible to mix Java and html.
  • At the top of the html page, replace the >!DOCTPYPE…< and the >html< tags with these 2 lines: 

        <!DOCTYPE html SYSTEM "http://www.thymeleaf.org/dtd/xhtml1-strict-thymeleaf-4.dtd">
        <html lang="en" xmlns="http://www.w3.org/1999/xhtml" xmlns:th="http://www.thymeleaf.org">
  • Now you can use the variable we added in the Model:
    <h1 th:text="${appName}">app</h1>
    The attribute th:text is a Thymeleaf Model attribute.
  • Restart the app in IntelliJ and refresh the browser
